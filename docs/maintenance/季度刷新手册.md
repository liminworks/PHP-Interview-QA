# 题库季度刷新手册

> 每季度末（3 / 6 / 9 / 12 月）执行一次，保持题库时效性与准确性。
> 预计用时：2-4 小时（视触发事件数量）。

---

## 触发事件速查

| 类别 | 触发条件 | 影响主题 |
|---|---|---|
| PHP 版本 | PHP 发布新 minor 版本（8.x） | PHP 语言篇 |
| 框架版本 | Laravel / Symfony 发布大版本 | Web 篇 |
| 数据库 | MySQL / Redis 发布重大版本 | 存储与中间件篇 |
| 安全公告 | CVE 影响 PHP 或常用扩展 | 安全篇 |
| 标准更新 | PSR / RFC 正式通过 | PHP 语言篇 / Web 篇 |
| 定期检查 | 每季度无论是否有触发事件 | 全部主题 |

---

## 执行流程

### Step 1 — 扫描待查证题目（≈15 分钟）

```bash
# 找出所有 needs_verification: true 的题目
grep -rl "needs_verification: true" docs/ --include="*.md"
```

逐一打开，对照官方文档或 RFC 确认内容准确性：
- ✅ 内容仍准确 → 将 `needs_verification` 改为 `false`，commit 注明来源
- ⚠️ 内容需更新 → 修正后改为 `false`，commit 说明变更点
- ❌ 内容已过时 → 重写或标注废弃

### Step 2 — 检查版本相关内容（≈30 分钟）

根据本季度触发事件，重点检查以下内容：

**PHP 新版本**

1. 搜索代码示例中是否有"PHP 7.x 才支持"等旧版本描述
   ```bash
   grep -r "PHP 7\." docs/ --include="*.md" -l
   ```
2. 检查 `docs/php/` 下追问链中涉及版本号的条目
3. 若新版本引入 breaking change，对受影响题目的「易错点」区块补充说明

**框架版本**

1. 检查 `docs/web/` 中 Laravel / Symfony 相关代码示例的 API 是否仍有效
2. 新版本新增重大特性时，在对应题目追问链末尾追加，或新建题目

**安全 CVE**

1. 检查 `docs/security/` 中的防御方案是否覆盖新漏洞向量
2. 若 CVE 影响现有「代码示例」中的写法，立即修正并在 commit 标注 CVE 编号

### Step 3 — 频率校准（≈20 分钟）

根据本季度面经讨论、招聘 JD 变化，调整频率标注：

```bash
# 查看全部频率标注分布
grep -r "^frequency:" docs/ --include="*.md" | sed 's/.*frequency: //' | sort | uniq -c
```

对频率与市场明显不符的题目，通过 ISSUE（`frequency_dispute` 模板）收集社区意见后再修改。

### Step 4 — 题量缺口分析（≈20 分钟）

```bash
# 统计各主题 × 难度的题量
python3 scripts/rebuild_all_indexes.py --dry-run 2>/dev/null || \
python3 - <<'EOF'
import os, re
from collections import defaultdict

counts = defaultdict(lambda: defaultdict(int))
for root, _, files in os.walk("docs"):
    for f in files:
        if f.endswith(".md") and f != "index.md":
            path = os.path.join(root, f)
            with open(path) as fh:
                content = fh.read()
            m = re.match(r'^---\n(.*?)\n---', content, re.DOTALL)
            if m:
                fm = dict(re.findall(r'^(\w+):\s*(.+)', m.group(1), re.M))
                topic = os.path.relpath(root, "docs")
                diff = fm.get("difficulty", "?")
                counts[topic][diff] += 1

for topic in sorted(counts):
    for diff in ["L1","L2","L3","L4","L5"]:
        n = counts[topic].get(diff, 0)
        if n > 0:
            print(f"  {topic:<25} {diff}: {n}")
EOF
```

对照 `主题目录.md` 中各主题，若某主题某难度 = 0，记录为「缺口」并在下一批次补充。

### Step 5 — 更新 CHANGELOG（≈10 分钟）

每次刷新完成后，在 `CHANGELOG.md` 顶部新增一个条目：

```markdown
## [刷新] YYYY-QX — 季度刷新

**刷新范围**：（触发事件说明）
**修改题数**：N 道
**新增题数**：N 道
**needs_verification 清零**：是 / 否（剩余 N 道）
```
